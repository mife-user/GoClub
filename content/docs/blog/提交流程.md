# 投稿与提交流程

如果你想给 GoClub 补充面试题、八股、配套文章，或者修正文档中的错误，可以按照下面这套流程提交。

PR 被成功合并后，站点会自动部署，你也会出现在贡献者页面。

## 1. Fork 仓库

先在 GitHub 上 Fork 这个仓库：

```text
https://github.com/LeoninCS/GoClub
```

## 2. Clone 到本地

把你 Fork 后的仓库克隆到本地：

```bash
git clone https://github.com/<your-github-name>/GoClub.git
cd GoClub
```

## 3. 新建分支

不要直接在 `main` 上改，先新建一个分支：

```bash
git checkout -b docs/your-topic
```

分支名可以按内容来起，比如：

```text
docs/add-tencent-interview
docs/fix-companion-links
docs/update-mysql-notes
```

## 4. 添加或修改文档

把内容放到 `content/docs` 对应的栏目下面：

- `content/docs/interview/`：面试真题
- `content/docs/baguwen/`：八股总结
- `content/docs/companion/`：配套文章
- `content/docs/blog/`：实践经验或技术记录

如果你是新增页面，记得同步修改对应目录下的 `_index.md`，把新文章加到列表入口里，不然页面可能虽然存在，但目录里看不到。

## 5. 本地预览

安装 Hugo 后，在项目根目录运行：

```bash
hugo server
```

然后打开：

```text
http://localhost:1313/
```

检查下面几件事是否正常：

- 页面能不能正常打开
- 目录页里能不能点到新文章
- 标题、图片、代码块、链接是否显示正常
- 有没有明显的错别字、断链或格式问题

## 6. 提交代码

确认无误后，提交你的改动：

```bash
git add .
git commit -m "docs: add your topic"
git push origin docs/your-topic
```

## 7. 发起 Pull Request

回到 GitHub，在你 Fork 的仓库页面发起 PR 到上游仓库的 `main` 分支。

PR 描述里建议简单写清楚：

- 这次新增或修改了什么内容
- 文档放在哪个目录
- 是否同步更新了对应的 `_index.md`

## 8. 等待合并与部署

PR 合并后，仓库会通过 GitHub Actions 自动构建并发布站点。

如果你的 PR 被成功合并，部署完成后，你会自动出现在贡献者页面中。

## 提交前小建议

- 文件名尽量清晰，方便后续检索
- 标题和目录命名尽量保持统一风格
- 能本地预览就尽量先预览一次
- 如果是补充同一栏目内容，尽量参考已有文章格式
